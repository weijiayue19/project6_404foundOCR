# 本地 OCR 图片与文档识别工具

一个基于 PaddleOCR 3.x 的本地 OCR 桌面工具，使用 Tkinter 提供图形界面，同时保留 `main.py` 命令行菜单用于验证图片核心流程。GUI 当前支持 JPG、JPEG、PNG、BMP 图片和 PDF 文档导入；快速识别使用 `PP-OCRv6_small`，深度识别使用 `PP-StructureV3`，识别结果、排版文本、缩略图和元数据统一写入 SQLite 历史库。

我们提供了详尽的项目文档，尤其是用户手册和系统设计报告。无论是理解系统设计，还是解决实际使用中遇到的问题，都可以在这些文档中找到想要的答案。

## 环境要求

- Python 3.11（推荐 3.11.9）
- Windows 或 macOS（暂不支持 Linux）

## 安装与启动

### 下载安装包

- macOS：前往仓库的 [Releases 页面](https://github.com/weijiayue19/project6/releases) 下载最新的 DMG 安装包并安装。
- Windows：安装包稍后上传，敬请期待。

macOS 安装包当前未进行 Apple 开发者签名。如果首次启动被系统拦截，可右键应用并选择“打开”，或前往“系统设置 → 隐私与安全性”选择“仍要打开”。

### 从源码启动

在项目根目录执行：

```bash
python3.11 -m venv .venv
source .venv/bin/activate   # Windows 使用：.venv\Scripts\activate
python -m pip install -r requirements.txt
python app.py
```

启动后会先进入识别方式选择页，可选择“快速识别”或“深度识别”；右下角也可以不进入工作区，直接打开历史查询。

如果只想验证图片核心流程，可运行命令行菜单：

```bash
python main.py
```

命令行菜单支持单张图片 OCR、批量图片 OCR、历史关键词查询、全部历史查看和按时间范围查询；PDF 文档识别目前走 GUI 深度识别路径。

## GUI 当前能力

- 支持通过文件选择、拖拽或剪贴板粘贴追加导入文件；再次添加不会覆盖现有队列。
- 快速模式导入 PDF 时会自动切换到深度识别；PDF 导入后先预览，用户点击“识别文档”后再开始 OCR。
- 图片可识别整图，也可在预览区拖拽框选后识别选区；多图模式下每张图独立保存选区。
- PDF 使用多页预览，支持滚轮、触控板和上下键滚动；PDF 不支持框选识别。
- 顶部“预处理”弹窗提供灰度化、自适应二值化和中值滤波去噪，默认关闭，可拖拽调整执行顺序。
- 缩略图可点击切换、右上角删除、长按拖动排序；识别和导出都会按当前缩略图顺序执行。
- 当前图片可执行旋转 90°、水平镜像和上下镜像；选区会随图片变换，旧识别结果会清除并要求重新识别。
- 单图识别在后台线程执行；批量识别在独立进程中按 FIFO 顺序逐张处理，并逐张回传进度，避免阻塞 Tkinter 界面。
- 识别工作区最小化为桌面小恐龙后会记住上次拖动位置，重启后继续使用；完成提示关闭或自动收起后会回到弹窗前位置。
- 选择页右上角的设置入口可启用或关闭小恐龙桌面助手，并可跳转到在线用户手册。
- 文件拖到小恐龙时会按扩展名自动拆分任务：图片任务使用快速识别，PDF 文档任务使用深度识别；混合拖拽会保持拖入顺序串行处理，必要时在后台创建另一个隐藏识别工作台，完成后点击查看会显示参与识别的窗口。
- 一轮宠物任务全部完成后，无论完成提示仍显示、手动关闭还是 10 秒后自动收起，都可以继续把下一批文件拖入普通宠物；每轮只启动进程识别新增或结果已失效的文件，已完成图片不会重复 OCR，历次批次的图片和结果会累计保留在对应快速/深度识别窗口。
- 工作区可打开独立的像素小恐龙小游戏窗口。
- 右侧结果区支持“全文拼接 / 按原位置排版”切换、复制、TXT 导出、清空和直接编辑。
- 编辑后的文本可通过保存按钮写入历史库，不覆盖模型 OCR 原文。
- 批量 TXT 导出支持“每张图片一个 TXT”或“按当前缩略图顺序合并为一个 TXT”。

当前桌面界面使用统一浅色像素风格：选择页模式卡片、工作区预览/结果区域、缩略图、历史查询窗口、弹窗和滚动条都使用项目自绘的像素风组件。

## 历史记录

成功识别后，记录会保存到本机 SQLite 数据库。源码运行时数据库位于 `data/ocr_records.db`；macOS 安装版位于 `~/Library/Application Support/404 Found OCR/ocr_records.db`。Windows 安装版发布后将使用 `%LOCALAPPDATA%\404 Found OCR\ocr_records.db`。历史记录包含：

- 原始路径、文件名、尺寸、文件大小和 SHA-256 哈希；
- 数据库内置 PNG 缩略图或 PDF 首页预览；
- 全文拼接文本、按原位置排版文本、OCR 文本块 JSON；
- 识别模式（快速识别 / 深度识别）和上传类型（图片 / 文档）；
- 用户最新编辑文本；
- 整图或选区坐标；
- 图片旋转 / 镜像状态、预处理摘要和耗时。

历史查询支持关键词、文件类型、识别模式和自定义日期范围筛选；详情区可在“全文拼接 / 按原位置排版”和“编辑前 / 编辑后”之间切换。选区记录会在历史预览中圈出区域，旋转或镜像后的记录会显示对应变换后的预览。历史记录编辑窗口支持选择并删除 SQLite 中的历史记录。

SQLite 是历史记录唯一数据源；当前代码不包含 `HistoryBST` 或 `src/history_manager.py`，GUI 和 CLI 均通过 `OCRSearchManager` 直接查询数据库。

## 快捷操作

- 将一张或多张 JPG、JPEG、PNG、BMP 图片或 PDF 文档从文件管理器拖入窗口；
- 复制图片后按 `Ctrl+V`，macOS 使用 `Command+V`，直接粘贴到队列；
- Windows 剪贴板中如果是图片或受支持文件列表，也会按文件方式导入；
- 图片载入后按 `Enter` 可直接识别整图，PDF 文档会触发“识别文档”；
- 最近点击图片区时，按 `Delete` 可删除当前图片；最近点击右侧文本框时，`Delete` / `Backspace` 保持文字编辑行为。

## Windows 兼容说明

程序默认使用 CPU 推理，并在入口处关闭 oneDNN/MKLDNN 优化，以规避部分 Windows 环境中的 `ConvertPirAttribute2RuntimeAttribute not support` 错误。请始终通过 `python app.py` 启动，确保兼容设置在 PaddleOCR 导入前生效。

程序也会在 Windows 上启用高 DPI 感知，尽量避免 Tkinter 界面在高分屏下发虚。

## 输入限制

- 图片格式：JPG、JPEG、PNG、BMP；
- 文档格式：PDF；
- 单文件大小上限：200 MB；
- 单张图片像素上限：5 亿像素；
- PDF 会校验文件存在、扩展名、大小，并用 `pypdfium2` 渲染页面预览；
- OCR 文本检测使用 `max` 策略限制长边 8192，并同步提高底层上限到 12288；
- 图片预览使用 Pillow 的 LANCZOS 高质量缩放算法。
